⚙️
The orchestrator pattern is most powerful for complex tasks — the LLM plans which specialist agents to use, coordinates them, and synthesises results.
// Agent Engine manages state, retries, and failures automatically
MODULE 04
Orchestration Patterns
Sequential pipeline: Each agent's output feeds the next. Simple, predictable, easy to debug. Best for linear workflows like data → analyse → report.
Parallel fan-out: Multiple agents work simultaneously on the same input. Best for independent analysis tasks where latency matters — legal, financial, and risk review in parallel.
Hierarchical: An orchestrator LLM dynamically plans which worker agents to use based on the task. Most flexible, highest autonomy, requires careful governance configuration.
Agent Engine: Gemini Enterprise's managed runtime handles agent state persistence, automatic retry on failure, and checkpoint recovery — critical for long-running workflows.

Grounding & RAG
Why grounding matters: LLMs hallucinate without grounding. RAG retrieves relevant documents at query time, giving the model accurate context rather than relying on training data.
Vertex AI Search: Managed RAG pipeline — ingest documents (PDF, HTML, structured data), automatic chunking and embedding, semantic search at query time. No custom vector DB needed.
Google Search grounding: Gives agents access to live web information. Responses include citations. Critical for tasks requiring current information beyond training cutoff.
Inline citations: Gemini Enterprise grounding responses include source citations — essential for enterprise compliance where agents must justify their answers.

MODULE 06
Governance Architecture
Agent Registry: Every agent must be registered before deployment. The registry stores metadata — purpose, permissions, owner, risk level — enabling discovery and accountability.
Agent Gateway: Runtime policy enforcement. All agent inputs and outputs pass through. Enforces data access controls, rate limits, output filtering, and PII detection.
Human-in-the-loop: Configure thresholds for mandatory human approval — amounts over $10k, sending to external parties, deleting data. Agents pause and wait for approval.
Audit logging: Complete audit trail stored in Cloud Logging. Every tool call, data access, and output logged with timestamps. Queryable for compliance and incident investigation.

    if amount > 5000:
        return {"status": "PENDING_HUMAN_APPROVAL",

    return payment_system.process(invoice_id, amount)

# Model can call multiple tools in one turn

results = await agent.run_parallel([

MODULE 07
Function Calling
Declarative tools: Define functions in Python with type hints and docstrings — Gemini automatically generates the JSON schema and decides when to call them based on user intent.
Parallel execution: Gemini 3.5 Flash supports parallel function calling — issuing multiple tool calls in a single model turn. Dramatically reduces latency for multi-system integrations.
Human-in-the-loop integration: Tools can return PENDING_HUMAN_APPROVAL to pause workflows for human review. The agent resumes automatically when approval is received.
Extensions vs Functions: Functions run in your code; Extensions are pre-built Google-managed integrations (Gmail, Calendar, Drive, BigQuery) with no custom code required.

MODULE 08
Agent Memory
In-context memory: The conversation history within a session. With 1M token context, Gemini 3.5 can maintain very long conversations without losing earlier context.
Session persistence: Stored in Firestore or Cloud SQL. Agents remember facts from previous conversations — user preferences, ongoing projects, previously established decisions.
Organisational knowledge base: Shared memory across agents in an organisation — standard processes, approval thresholds, terminology. Agents learn once, apply everywhere.
Memory scoping: Privacy controls determine what each agent can access. User memory is private; team memory is shared with the team; org memory is organisation-wide.

// Data handling principles:
No training on your data CMEK encryption VPC-SC support Data residency controls GDPR / NDPA compliant
MODULE 10
Security Architecture
Agent identity: Each agent runs as a dedicated GCP service account with the minimum IAM permissions required. Compromise of one agent cannot escalate to broader system access.
CMEK (Customer-Managed Encryption Keys): Organisations control their own encryption keys for all data processed by Gemini Enterprise. Google cannot decrypt without your key.
VPC Service Controls: Isolate Gemini Enterprise within your VPC perimeter. Prevents data exfiltration even from authorised users and reduces blast radius of compromised credentials.
Data residency: Process data in specified regions. Key for Nigerian NDPA, EU GDPR, and other data localisation regulations. Configurable at project and resource level.

// Example monthly cost — 100-person team, 50 agent tasks/day:
100 users × 50 tasks × ~500 tokens avg = 2.5M tokens/day
30 days × $0.075/M = ~$5.60/day
+ Agent Engine runtime: ~$50/month
Total: ~$220/month for 100-person team
MODULE 11
Cost Optimisation
Context caching: If your agent uses the same large document or system prompt repeatedly, cache it. Cached token queries cost 75% less than uncached — the biggest single cost lever.
Model selection: Flash is 10x cheaper than Gemini Pro for comparable tasks. Reserve Omni for genuinely multimodal workflows. Most enterprise text tasks are well-served by Flash.
Batch API: For non-real-time processing (overnight reports, bulk document analysis), use the Batch API at 50% discount vs online inference.
Cost monitoring: Set budget alerts in Cloud Billing. Use agent-level labels to track cost per team, per workflow, per agent — critical for chargeback and ROI measurement.

Implementation Roadmap
Start with governance: Set up Agent Registry, Gateway policies, and IAM before building any agents. Retrofitting governance is 10x harder than starting with it.
First agent principle: Choose a low-risk, high-repetition task for your first agent — document summarisation, data extraction, or meeting notes. Build confidence before complex workflows.
Measure from day one: Define success metrics before deployment — time saved, accuracy rate, cost per task. ROI measurement drives organisational buy-in for expansion.
Agent lifecycle: Treat agents like software — version control, testing, staged rollout, monitoring, deprecation. The ADK and Agent Engine provide all necessary infrastructure.
